一句话

这不是爬虫，是小卖家选品决策流水线

系统把“市场机会、数据可信度、执行适配、广告压力、历史经验”拆开判断，再用后端数学安全阀兜住误判。
核心目标：推荐进入人工验证，而不是替代人工下单
全链路

从触发到沉淀：一个闭环

触发定时任务或手动启动
›
枢纽读策略与记忆，生成 Brief
›
鹰眼调用 ProductSelector，输出 raw
›
熔炉审计评分，输出 scored
后端棱镜安全阀：发布 / 回环 / 换方向
›
枢纽发布研报，写公共日记
›
记忆下轮自动召回经验与避坑
›
下一轮更少重复，更快收敛
状态机：PENDING_BRIEF → BRIEFING → SCOUTING → SCORING → EVALUATING → PUBLISHING → DONE
三位 Agent

权限分离，让每个判断更干净

破壁_枢纽

战略协调者。创建 SelectionBrief，发布最终研报，写入选品公共日记本。

破壁_鹰眼

研究取证员。只负责调用 ProductSelector，产出最小可判断证据包。

破壁_熔炉

证据审判员。零工具运行，只读 raw，做 Hard Gates、四分制和终态裁决。

后端负责状态流转、锁、回环、归档和数学安全阀
记忆能力

公共日记本是系统的长期选品脑

它记什么

  • [淘汰] 最近失败方向与具体死因
  • [待观察] 预筛过线但本轮未深挖的机会
  • [经验] 跨产品可复用的避坑规则和成功模式

它不是什么

  • 不是销量、PPC、CVR、FBA 的证据来源
  • 不能替代本轮 ProductSelector 取证
  • 不能把一个月前的失败永久拉黑整个大类
记忆入口：选品公共日记本 + TagMemo + AIMemo

策略创建

枢纽先把“模糊选品”变成可验证 Brief

输入

  • Strategy Profile 自动注入
  • 公共日记本历史经验
  • 当前 run_id 与宽泛探索规则

输出

  • 研究假设、目标场景、目标客户
  • 2-3 个真正不同的候选方向
  • 英文种子词、价格带、排除红线
  • 给鹰眼的 Level-1 / Level-2 取证任务
枢纽提出假设，不直接做商业推荐
鹰眼取证

先便宜横扫，再只深挖最优方向

L1

方向体检

关键词需求、转化率、产品选品、类目收敛、可比标杆初判。

L2

候选验证

竞品、FBA 样本、反查词、商品页、评论痛点，只给有潜力方向；可比性缺口要做 ≥2 个真同类标杆平行反查。

L3

回环补采

只补熔炉指定字段，保留旧 raw，禁止泛泛“继续调研”。

目标：Min Evidence Pack，而不是堆满不可读数据
可比性

不能让错误标杆污染经济性结论

风险

Top ASIN 可能只是流量高，但形态、价格带、成熟度都不可比。拿它当标杆会高估自然流量和广告容错。

守卫

鹰眼输出 comparable_anchors，熔炉禁止使用 not_comparable 标杆进入综合分；自然流量红利不能单锚自证。

anchor_reverse_verified = ≥2 个真同类 comparable ASIN 反查共同支撑
争议 Top1 不能作为唯一证据；细分类目 nodeIdPath 用来补足同类盘面
熔炉审判

先 Hard Gates，再输出四类分数

Hard Gates 一票否决

  • 侵权、医疗补剂、服装鞋帽、禁售
  • 超大件、重货、高认证、高售后
  • 负贡献利润、资金不可承受

人工验证项不等于硬门

  • BOM 询价、专利排雷、真人打样
  • Seller Central FBA 最终实测
  • 写入 Next Validation Plan 与 Kill Criteria
RECOMMEND = 值得进入人工验证 / 立项验证，不是直接下单生产
四分制

把“机会好不好”拆成三个问题和一个排序

OpportunityScore

市场本身有没有机会：需求、增长、差异化、进入窗口。

DataReliabilityScore

我们有多确定：来源、样本、字段完整、多源一致。

ExecutionFitScore

小卖家能不能做：资金、供应链、上架、售后、迭代。

FinalScore

熔炉业务排序参考分，用于推荐档位和报告表达。

市场不错但关键假设未闭合，且有正期权价值，才进入待观察

利润与广告

系统看的是单件贡献利润，不是热度

UnitContribution

售价 - 平台佣金 - BOM - 头程 - FBA - 包装 - 退货预留 - 优惠券 - 仓储预留。

保守压力测试

SellerSprite CVR 会折减；PPC 会放大；广告派生字段失真时降置信，不单独硬淘汰。

paid_traffic_ratio < 0.60 只接受 anchor_reverse_verified + ≥2 个 source_asins
单锚、不可比 ASIN、或无 basis 下调都会被钳回 0.60 并标失真
卖家优势

Listing 场景代入能力被显式数学化

适合放大

摆件、玩具、礼品、家居氛围、强场景图能影响购买决策的产品。

不适合放大

锤子、线缆、替换件、规格和功能主导的产品，Listing 优势有限。

diff_effective = diff_base + L × seller_skill × (1 - diff_base) × gain
熔炉输出 listing_leverage_score；后端用它影响差异化支柱
区间决策

不是单点切割，而是点估计 ± 不确定带

< 42

乐观都不及格

预算内 DROP_AND_RESELECT；硬门永远优先。

中间

交给业务裁决

由熔炉 verdict 决定推荐或待观察；WATCHLIST 只给有正期权价值且关键假设未闭合的方向。

≥ 62

悲观也过线

熔炉 RECOMMEND 可以成立，进入人工验证。

后端防误杀、防数学错误；中间区间不静默淘汰，但也不自动塞进观察池
回环守卫

回环只补关键缺口，禁止无限研究

允许

  • Critical Gap 或最多一次 Important Gap
  • 指定字段、工具、关键词或 ASIN
  • 高价值候选缺 ≥2 个真同类标杆反查证据

禁止

  • 同字段已补采，或已在 unfetchable_gaps
  • 只写“继续调研”但没有具体请求
  • 超过 global / scout 回环上限
广告洗白类失真无法回环治愈时，终局降档为待观察发布

最终研报

给人看的不是一个分数，而是一条证据链

必须并列展示

  • 熔炉业务四分
  • 后端棱镜点估计与区间
  • 利润、CVR、PPC、CPA 压力测试
  • recommendation_tier.display 中文强度

必须给出行动边界

  • 缺口与假设
  • 可比 / 不可比标杆与自一致检查
  • Kill Criteria
  • Next Validation Plan
  • 本轮淘汰记录与经验标签
报告用于“下一步怎么验证”，不是让读者盲信单一分数
记忆复利

每一轮结束，系统都会变得更不重复

  • [淘汰] 记录具体死因和已试关键词，让下轮避开重复探测。
  • [待观察] 保存过线但未深挖方向，让下轮优先重拾机会池。
  • [经验] 沉淀跨产品规律，例如重量、退货、认证、广告倒挂阈值。
  • Tag 行 把状态、产品核心词、死因或机会标签变成可检索入口。
这就是 agent 记忆对整套系统的真实作用：降重复、提早停、稳裁决、促收敛
系统边界

它负责高置信参考，不负责替你承担商业风险

能做

快速筛掉明显不适合方向，浮现值得人工验证的机会。

要谨慎

SellerSprite 广告派生字段、冷门长尾词、自然流量假设。

仍需人工

BOM 询价、专利排雷、真人打样、Seller Central 最终 FBA 实测。

系统的强项是把“该验证什么”讲清楚
演示收束

可以这样讲给团队听

  • 枢纽负责把策略与记忆转成可抓取 Brief。
  • 鹰眼负责用最少工具拿到可判断证据，并显式标记可比 / 不可比标杆。
  • 熔炉负责业务裁决，先硬门再四分，再做 self_consistency_check。
  • 后端棱镜负责数学审计、区间决策、回环守卫，以及广告洗白降档。
  • 公共日记本负责把每轮结果变成下轮可召回的经验。
一句话：让选品从“灵感 + 表格”变成“证据 + 记忆 + 安全阀”
