# AmazonAds ToolBox (Amazon Ads 工具箱)

`AmazonAds` 通过 Amazon Ads MCP 配置读取 Amazon Ads 账户结构和报表数据。这个工具箱是中立的（不绑定具体账户）。在用户配置他们自己的账户之前，请使用占位符。

## 安全要求 (Safety)

- 不要泄露 access tokens、refresh tokens、client secrets、MCP headers 或签名的报表下载 URL。
- 不要假设存在默认的卖家账户。请询问用户以配置或确认 `accountName`、`profileId` 和 `marketplace`。
- 对于同一个账户、日期范围、报表类型和字段集，不要创建重复的报表。
- 写操作必须首先使用 `propose_*` 生成建议，并在执行 `apply_*` 之前要求用户明确批准，同时需要插件操作密码。

## 默认占位符 (Default Placeholders)

```text
accountName: <YOUR_ACCOUNT_NAME>
adsAccountId: <YOUR_ADS_ACCOUNT_ID>
profileId: <YOUR_PROFILE_ID>
entityId: <YOUR_ENTITY_ID>
marketplace: US
```

## 直接查询结构 (Direct Structure Queries)

当用户询问当前配置了什么时，使用以下命令：

- `list_campaigns`
- `list_ad_groups`
- `list_ads`
- `list_targets`
- `check_product_eligibility`
- `get_billing_status`
- `list_account_users`

示例：

```text
<<<[TOOL_REQUEST]>>>
tool_name:「始」AmazonAds「末」,
command:「始」list_campaigns「末」,
accountName:「始」<YOUR_ACCOUNT_NAME>「末」,
marketplace:「始」US「末」,
maxResults:「始」50「末」
<<<[END_TOOL_REQUEST]>>>
```

直接查询结构的命令不能回答历史表现或数据问题。

## 报表查询 (Report Queries)

使用报表来获取曝光量 (impressions)、点击量 (clicks)、花费 (cost)、销售额 (sales)、订单量 (orders)、ACOS、ROAS、广告位表现 (placements)、投放表现 (targets) 和搜索词 (search terms)。

推荐的高级命令：

```text
<<<[TOOL_REQUEST]>>>
tool_name:「始」AmazonAds「末」,
command:「始」get_report_data「末」,
reportKind:「始」campaign「末」,
dateFrom:「始」2026-01-01「末」,
dateTo:「始」2026-01-07「末」,
accountName:「始」<YOUR_ACCOUNT_NAME>「末」,
marketplace:「始」US「末」,
mode:「始」get_or_create「末」
<<<[END_TOOL_REQUEST]>>>
```

常见的 `reportKind` 值：

- `campaign`
- `placement`
- `target`
- `search_term`
- `keyword`

有用的归档管理命令：

- `list_report_jobs`
- `read_report_artifact`
- `export_report_artifact`
- `aggregate_report_archives`

## 报表轮询规则 (Report Polling Rule)

当创建或查询报表时，如果返回的状态是 `PENDING` 或 `PROCESSING`，你**必须立刻停止工具调用并结束当前回复**，告诉用户报表正在生成中，然后**等待用户的下一次命令**。哪怕你觉得需要等待，也坚决不要自己陷入无限循环的轮询中，务必等待用户发起下一次交互后再去查询。

## 字段探测规则 (Field Probe Rule)

仅在用户要求、Amazon 拒绝了某个字段，或模式不确定性阻碍了任务时，才探测字段。保持探测批次尽量小。

```text
<<<[TOOL_REQUEST]>>>
tool_name:「始」AmazonAds「末」,
command:「始」probe_report_fields「末」,
preset:「始」campaign_basic「末」,
candidateFields:「始」budgetCurrency.value,metric.sales「末」,
days:「始」1「末」
<<<[END_TOOL_REQUEST]>>>
```

## Token 刷新 (Token Refresh)

仅在启用了插件 Token 刷新且用户要求时，或发生 401/Token 错误并且 `get_status` 显示刷新功能可用时，才调用 `refresh_token`。

如果 `tokenRefresh.enabled=false`，请告诉用户检查外部的 Token 刷新器或 MCP 配置。
