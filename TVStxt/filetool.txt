四.文件管理
# 物理环境判别准则：
## 凡是涉及 VCP 系统逻辑、插件配置、日志文件、/home/your-user/ 路径的操作，均判定为【服务器后端】。
## 凡是涉及 历史记录.json、E:\ 盘符、或用户明确提到的“本地”操作，均判定为【本地前端】。

1.文件搜索（环境差异强协议）。
- 搜索【Windows本地前端】(路径如 E:\VCPchat\...)，使用 tool_name:「始」LocalSearchController「末」。该工具依赖 Windows Everything HTTP Server，query 支持 Everything 语法。
- 搜索【Ubuntu服务器后端】(路径如 /home/your-user/...)：当前 ServerSearchController 仍是 Everything HTTP 兼容实现，会访问服务器本机 http://127.0.0.1:8025；除非已确认服务器端部署了兼容服务，否则不要调用 ServerSearchController。
- 服务器端搜索优先策略：已知路径直接用 ServerFileOperator 的 ReadFile/FileInfo；已知目录用 ServerFileOperator 的 ListDirectory/SearchFiles；搜索 VCP 源码内容用 ServerCodeSearcher；必须执行 Linux 命令时才用 LinuxShellExecutor。
- 查询 /home/your-user/、Plugin/、TVStxt/、config.env、日志文件等服务器文件时，禁止调用 LocalSearchController，也不要用 LinuxShellExecutor 代替文件读取/搜索。
  tool_name:「始」LocalSearchController「末」, // 仅限 Windows 本地前端；ServerSearchController 仅在服务器端兼容服务已确认可用时使用
command:「始」search「末」,
query:「始」VCP a.txt「末」, // LocalSearchController 的语法和 Everything 一致；服务器端默认不要假设 Everything 可用。
maxResults:「始」50「末」

20.[重要]如何创建“内部内容有「始」，「末」”的文件！使用转义文件创造插件，它会自动将文本内的「始exp」，「末exp」，<<<[TOOL_REQUEST_EXP]>>>，<<<[END_TOOL_REQUEST_EXP]>>>进行替换，从而安全创造包含始末的文件。
tool_name:「始」FileOperator「末」,
command:「始」WriteEscapedFile「末」,
filePath:「始」/path/to/new_plugin/plugin-manifest.json「末」,
content:「始」{
  "description": "...调用格式:\n<<<[TOOL_REQUEST_EXP]>>>\ntool_name:「始exp」MyNewPlugin「末exp」,\n...\n<<<[END_TOOL_REQUEST_EXP]>>>"
}「末」

[重要提示：事实上，在你向用户演示指令，而非真正调用工具的时候，也应该遵守exp转义原则！]

21.命令行工具（环境路由强协议）。
PowerShellExecutor/ServerPowerShellExecutor 是 Windows PowerShell 工具，会启动 powershell.exe；Ubuntu 服务器后端不要使用 ServerPowerShellExecutor。
直接查看/编辑文件时优先使用 FileOperator/ServerFileOperator，只有确实需要执行系统命令时才调用 Shell 工具。
【环境路由强协议】：
- 操控【Ubuntu服务器后端】(Linux Shell)，必须使用 tool_name:「始」LinuxShellExecutor「末」
- 操控【Windows本地前端】(带GUI窗口)，才使用 tool_name:「始」PowerShellExecutor「末」
- LinuxShellExecutor 必须显式传 hostId；操作当前 Ubuntu 服务器一律使用 hostId:「始」local「末」。除非用户明确指定远程资产并确认 SSH 可用，不要选择 DB1、DE-server 等发现列表里的 SSH 主机。
- LinuxShellExecutor 不用于读取/搜索文件；如确需执行只读命令，避免使用重定向符号、2>&1、后台 &、分号 ;。
- LinuxShellExecutor 只读检索命令应优先写成安全管道，例如 `grep -rn "关键词" /path/to/VCPToolBox/Plugin/ | head -n 100`、`find /path/to/VCPToolBox/Plugin -type f -mmin -120 -maxdepth 2 | head -n 100`。不要使用 `2>/dev/null` 或 `|| true`。
tool_name:「始」LinuxShellExecutor 或 PowerShellExecutor「末」, // 严格按上述物理环境选择
command:「始」ls -la /path/to/VCPToolBox「末」,
hostId:「始」local「末」, // LinuxShellExecutor 必填；当前服务器固定 local
timeout:「始」60000「末」,
outputFormat:「始」formatted「末」

22.代码检索器。
调用此工具在 VCP 服务器后端源码中搜索指定的代码片段或关键词。支持使用正则表达式进行高级搜索。通过这个插件更快理解 VCP 服务器工作原理！搜索 `server.js`、`Plugin.js`、`modules/`、`routes/`、`Plugin/`、`TVStxt/` 等 VCP 项目文件时，优先使用本工具，不要改用 LinuxShellExecutor。
参数:
- query (字符串, 必需): 需要搜索的关键词、代码片段或正则表达式。
- search_path (字符串, 可选): 指定要搜索的相对路径。如果未提供，则默认为整个项目的工作区。
- case_sensitive (布尔值, 可选, 默认false): 是否区分大小写搜索。
- whole_word (布尔值, 可选, 默认false): 是否进行全词匹配。
- context_lines (整数, 可选, 默认2): 返回结果时，在匹配行前后额外显示的上下文行数。
tool_name:「始」ServerCodeSearcher「末」,   // VCP 后端源码搜索固定使用 ServerCodeSearcher；不要写 CodeSearcher
query:「始」关键词或正则表达式「末」,
search_path:「始」(可选)相对路径「末」,
case_sensitive:「始」(可选)true/false「末」,
whole_word:「始」(可选)true/false「末」,
context_lines:「始」(可选)数字「末」

示例：
<<<[TOOL_REQUEST]>>>
tool_name:「始」ServerCodeSearcher「末」,
query:「始」admin_api/server/restart「末」,
search_path:「始」Plugin「末」,
context_lines:「始」2「末」
<<<[END_TOOL_REQUEST]>>>
